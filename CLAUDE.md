# ルール

## プロジェクト構成

- 再利用可能なものはなるべく、 `partials/components` に切り出すこと
    - `partials/components` に切り出したコンポーネントのスタイルは `assets/css/` に同名のファイルとして配置すること
    - `screen.css` に追記はしないこと
- 開発サーバーは常に起動しているため、確認する際は、 `playwright mcp` を利用すること

## CSS

- CSS はグローバルスコープになるため、prefix として css ファイルのファイル名を使用すること
- 次のプロパティは CSS 変数を定義しているため、 `assets/css/variables.css` の値を参照すること
    - `font`
    - `letter-spacing`
    - `line-height`
    - `padding`
    - `gap`
    - `border-radius`
    - `color`
    - `background-color`
- サイトのベースとなる `font-family` は個々の要素に記述しないこと
    - 部分的に異なるフォントを利用する場合にのみ、個々の要素に `font-family` を指定すること
- コンポーネントを作る時は`props`に`class`を含めないこと
- レイアウトを記述するときは `grid` を最優先とすること
- メディアクエリを使用する場合は、必ず SP をベースとして指定してください。
    - ページ幅に応じてスタイルが変化する場合は、メディアクエリを使用してください。
    - 親要素の幅に応じてスタイルが変化する場合は、コンテナクエリを使用してください。
    - 指示がない限り、メディアクエリを実装しないでください。
